=== CHAPTER 7: INPUT/OUTPUT (I/O) ===

[1. External Devices & I/O Module Interface]

Interface to System Bus: คุยกับ CPU ผ่าน 3 สายหลัก: Address Lines (ระบุอุปกรณ์), Data Lines (ส่งข้อมูล), Control Lines (คำสั่ง Read/Write)

Interface to External Device: คุยผ่านสัญญาณ Data, Status (Ready/Not Ready), และ Control (สั่งอุปกรณ์ทำงาน)

Internal Structure (Intel 82C55A Model):

Data Registers: พักข้อมูล (Buffer)

Status/Control Registers: เก็บสถานะอุปกรณ์และรับคำสั่งการตั้งค่าจาก CPU

I/O Logic: ตัวถอดรหัส (Decoder) ว่า Address ที่ส่งมาเป็นของอุปกรณ์ตัวไหน และต้องทำอะไร

[2. IRA Control Characters - Full Bit Breakdown]

Format Control (ควบคุมหัวพิมพ์/Cursor):

BS (0001000): ถอยหลัง 1 ตำแหน่ง

HT (0001001): Tab แนวนอน

LF (0001010): ขึ้นบรรทัดใหม่

CR (0001101): กลับต้นบรรทัด

Transmission Control (สัญญาณสื่อสาร):

ENQ (0000101): Enquiry ถามตัวตน "คุณคือใคร/พร้อมไหม"

ACK (0000110): ยืนยันข้อมูลถูกต้อง

NAK (0010101): ปฏิเสธข้อมูล (Error) ให้ส่งใหม่

SYN (0010110): ปรับจังหวะเวลา (Idle fill)

Miscellaneous: BEL (เสียงเตือน), ESC (เปลี่ยนความหมายอักขระถัดไป), CAN (Cancel ยกเลิกข้อมูล)

[3. I/O Control Techniques - Operational Flow]

Programmed I/O: CPU ต้องเช็คสถานะในลูป (Wait Loop) จนกว่าบิต Status จะเป็น Ready

Interrupt-driven I/O (Detailed Hardware Steps):

อุปกรณ์ส่งสัญญาณ Interrupt

CPU ประมวลผลคำสั่งปัจจุบันให้จบ

CPU ตรวจสอบ Priority และส่ง ACK

Push: บันทึก PSW (Flags) และ PC (ที่อยู่ถัดไป) ลง Control Stack

Load PC: โหลดที่อยู่ของ ISR (Interrupt Service Routine) มาทำงาน

Execute ISR: ประมวลผลการย้ายข้อมูล

Pop: ดึงค่าเดิมกลับมาทำงานต่อ

DMA (Direct Memory Access):

Cycle Stealing: DMA จะ "ขโมย" บัสในช่วงเวลาสั้นๆ (1 Clock) หลังจังหวะ Fetch/Execute เพื่อส่งข้อมูล โดย CPU จะหยุดชะงัก (Suspended) ชั่วคราว

Configurations: - Detached DMA: CPU โดนเบรก 2 รอบ (I/O->DMA และ DMA->Mem)

Integrated DMA: CPU โดนเบรก 1 รอบ (DMA->Mem) ข้อมูลจาก I/O เข้า DMA โดยตรง

[4. External Interconnection Standards]

USB-C Pin Details: - VBUS: จ่ายไฟ

TX/RX: ส่ง/รับความเร็วสูง (SuperSpeed)

CC1/CC2: ตรวจสอบการเสียบหงาย/คว่ำ และเจรจาการจ่ายไฟ (Power Delivery)

D+/D-: รองรับ USB 2.0 รุ่นเก่า

Ethernet Evolution: จาก Bus Topology (ชนกันง่าย) -> Star Topology (Switch) ที่ใช้การจำ Address ประจำพอร์ต ทำให้คุยพร้อมกันได้แบบ Full Duplex โดยไม่กวนกัน

=== CHAPTER 8: CPU & INSTRUCTION SETS ===

[1. Integer Representation & Overflow Box]

Twos Complement Range (n บิต): $-2^{n-1}$ ถึง $2^{n-1}-1$

Value Box Calculation: การแปลงฐาน 2 เป็นฐาน 10 โดยบิตซ้ายสุด (MSB) มีค่าน้ำหนักเป็นค่าติดลบ (เช่น 8-bit MSB = -128)

Overflow Rule: ถ้า (+) + (+) = (-) หรือ (-) + (-) = (+) แสดงว่าเกิดการล้นของบิต

Hardware Adder/Subtractor: ประกอบด้วย A, B Registers, Complementer (Inverter + 1), และ Switch ที่เลือกว่าจะส่งค่าปกติหรือค่า Complement เข้า Adder

[2. Booth's Algorithm (The Multiplication Logic)]

Input: $M$ (Multiplicand), $Q$ (Multiplier)

Check Bits ($Q_0, Q_{-1}$):

10: $A = A - M$ (บวกด้วย Twos Complement ของ M)

01: $A = A + M$

11 หรือ 00: ไม่ต้องทำอะไร

Action: หลังเช็คบิต ต้องทำ ASR (Arithmetic Shift Right) ที่ $A, Q, Q_{-1}$ โดยต้องคัดลอกบิต MSB ของ A เดิมไว้เพื่อรักษาเครื่องหมาย

[3. Floating-Point (IEEE 754 Single Precision)]

Structure (32-bit): [Sign: 1] | [Biased Exponent: 8] | [Fraction: 23]

Bias = 127: ค่า Exponent จริง = ค่าที่เก็บในบิต - 127

Implicit Bit: เลข 1 หน้าจุดทศนิยมที่ไม่ได้เก็บจริง ($1.fraction$)

Density Gap: ระยะห่างระหว่างตัวเลขสองตัวที่ติดกัน $\Delta X = 2^{(Exponent-127-23)}$ ตัวเลขจะยิ่งห่างกันมาก (แม่นยำน้อยลง) เมื่อค่าเพิ่มสูงขึ้น

Arithmetic Flow: ต้องจัด Exponent ให้เท่ากันก่อนบวกลบ โดยการเลื่อน Significand ของตัวที่มี Exponent "น้อยกว่า" ไปทางขวา

[4. Machine Instructions & Address Modes]

Elements: Opcode, Source/Result Operands, Next Instruction Reference

Address Optimization:

3-Address: สั้นสุด ($Y = A + B$) แต่ซับซ้อนและเปลือง Register

1-Address: ใช้ Accumulator ($AC$) เช่น ADD B หมายถึง $AC = AC + B$

Instruction Cycle State: Fetch -> Address Calculation -> Operand Fetch -> Data Operation -> Store Result -> Fetch Next

Design Trade-off: More Addresses = โปรแกรมสั้นแต่ฮาร์ดแวร์แพง, Fewer Addresses = โปรแกรมยาวแต่ CPU รันได้ไวและประหยัดงบ
