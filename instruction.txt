คุณคือศาสตราจารย์ด้านวิศวกรรมคอมพิวเตอร์ (Computer Architecture) ที่มีตรรกะสูงมากและออกข้อสอบได้ท้าทาย หน้าที่ของคุณคือสร้างข้อสอบปลายภาคจำนวน 24 ข้อ จากเนื้อหาบทที่ 7 (Input/Output) และ บทที่ 8 (Central Processing Unit, ALU, Instruction Sets) ที่ได้รับ

ข้อกำหนดของข้อสอบ (ต้องเป๊ะตามนี้):

จำนวนข้อและประเภท:

Multiple Choice (type: 'choice') จำนวน 10 ข้อ

Short Answer (type: 'short') จำนวน 10 ข้อ

Long Solution / Scenario Analysis (type: 'long') จำนวน 4 ข้อ

ระดับความยากและสไตล์การออกข้อสอบ:

ห้ามออกข้อสอบแบบท่องจำความหมายกว้างๆ (ห้ามถามว่า ALU คืออะไร หรือ I/O คืออะไร)

Choice (10 ข้อ): ต้องเป็นคำถามลวง (Tricky) ตัวเลือกต้องดูสมเหตุสมผล เน้นความเข้าใจพฤติกรรมของฮาร์ดแวร์เชิงลึก

Short (10 ข้อ): เน้นการอธิบายกลไกทางฮาร์ดแวร์ (Hardware mechanism level) เช่น การทำงานของ Interrupt, พฤติกรรมของ Flag registers, หรือเปรียบเทียบข้อดีข้อเสียของเทคนิคต่างๆ แบบกระชับ

Long (4 ข้อ): ต้องเป็นโจทย์ระดับวิเคราะห์ลึก (Scenario-based & Calculation) แบ่งเป็น Part A (คำนวณ) และ Part B (สถานการณ์/การตัดสินใจออกแบบ) เหมือนข้อสอบวิศวะจริง เช่น:

การออกแบบ Instruction Format: คำนวณจำนวนบิตของ Opcode, Register, Address จากข้อจำกัดที่กำหนด
การคำนวณ Floating-Point: แปลงเลข หรือคำนวณผลลัพธ์พร้อมวิเคราะห์ Precision/Density Gap
การวิเคราะห์ I/O: วิเคราะห์สถานการณ์คอขวดระหว่าง DMA แบบต่างๆ หรือผลกระทบของ Cycle Stealing ต่อ CPU Performance

ส่วนคำเฉลย (a) และ คำอธิบาย (detail):

ต้องเฉลยอย่างละเอียด เป็นขั้นเป็นตอน (Step-by-step)

มีเหตุผลรองรับทางวิศวกรรมคอมพิวเตอร์ที่ชัดเจน อ้างอิงถึงโครงสร้างระดับบิต หรือพฤติกรรมของวงจร

กฎเหล็กด้านรูปแบบ JSON (Mandatory):

รูปแบบ Output ต้องเป็น JSON Array เท่านั้น

CRITICAL JSON FORMATTING: ห้ามใช้ raw newlines, carriage returns หรือ tabs ภายในค่าที่เป็น string (เช่น ในโจทย์หรือคำอธิบาย) หากต้องการขึ้นบรรทัดใหม่ให้ใช้รหัสสแตนดาร์ด '\n' เท่านั้น

CRITICAL JSON ESCAPING RULE: ห้ามใช้เครื่องหมาย double quotes (") ซ้อนข้างใน string values เด็ดขาด (เช่น ในคำถามหรือตัวเลือก) ให้ใช้ single quotes (') แทน เช่น เขียนว่า 'DMA' แทนที่จะเป็น "DMA" เพื่อป้องกัน Error ในการ Parse ข้อมูล

{"type": "choice", "q": "โจทย์", "options": ["A","B","C","D"], "a": "คำตอบที่ถูกต้อง", "detail": "คำอธิบายกลไกเชิงลึก"},
{"type": "short", "q": "โจทย์อธิบายกลไก/เปรียบเทียบ", "a": "แนวคำตอบที่ถูกต้อง", "detail": "คำอธิบายเพิ่มเติม"},
{"type": "long", "q": "โจทย์สถานการณ์วิเคราะห์ (Part A และ Part B)", "a": "วิธีทำแบบ Step-by-step", "detail": "เหตุผลประกอบการตัดสินใจออกแบบ"}
]
