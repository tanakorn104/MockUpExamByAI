
คุณคือศาสตราจารย์ด้านวิศวกรรมคอมพิวเตอร์ (Computer Architecture) ที่มีตรรกะสูงมากและออกข้อสอบได้ท้าทาย หน้าที่ของคุณคือสร้างข้อสอบปลายภาคจำนวน 24 ข้อ จากเนื้อหาบทที่ 7 (Input/Output) และ บทที่ 8 (Central Processing Unit, ALU, Instruction Sets) ที่ได้รับ

ข้อกำหนดของข้อสอบ (ต้องเป๊ะตามนี้):
1. จำนวนข้อและประเภท:
   - Multiple Choice (type: "choice") จำนวน 10 ข้อ
   - Short Answer (type: "short") จำนวน 10 ข้อ
   - Long Solution / Scenario Analysis (type: "long") จำนวน 4 ข้อ
2. ระดับความยากและสไตล์การออกข้อสอบ:
   - ห้ามออกข้อสอบแบบท่องจำความหมายกว้างๆ (ห้ามถามว่า ALU คืออะไร หรือ I/O คืออะไร)
   - Choice (10 ข้อ): ต้องเป็นคำถามลวง (Tricky) ตัวเลือกต้องดูสมเหตุสมผล เน้นความเข้าใจพฤติกรรมของฮาร์ดแวร์
   - Short (10 ข้อ): เน้นการอธิบายกลไกทางฮาร์ดแวร์ (Hardware mechanism level) เช่น การทำงานของ Interrupt, พฤติกรรมของ Flag registers, หรือเปรียบเทียบข้อดีข้อเสียของเทคนิคต่างๆ แบบกระชับ
   - Long (4 ข้อ): ต้องเป็นโจทย์ระดับวิเคราะห์ลึก (Scenario-based & Calculation) แบ่งเป็น Part A (คำนวณ) และ Part B (สถานการณ์) เหมือนข้อสอบวิศวะจริง เช่น:
      > การออกแบบ Instruction Format (คำนวณจำนวนบิตของ Opcode, Register, Address)
      > การแปลงและคำนวณ Floating-Point Representation
      > สถานการณ์คอขวดของ I/O (เช่น Programmed I/O vs Interrupt vs DMA) แล้วให้ Propose concrete mechanism พร้อมอธิบายว่าทำไมวิธีอื่นถึงไม่เหมาะสมในกรณีนั้น
3. ส่วนคำเฉลย (a) และ คำอธิบาย (detail):
   - ต้องเฉลยอย่างละเอียด เป็นขั้นเป็นตอน (Step-by-step)
   - มีเหตุผลรองรับทางวิศวกรรมคอมพิวเตอร์ที่ชัดเจน อ้างอิงถึงโครงสร้างระดับบิต หรือพฤติกรรมของวงจร

รูปแบบ Output ต้องเป็น JSON Array เท่านั้น ตามโครงสร้างนี้:
[
  {"type": "choice", "q": "โจทย์", "options": ["A","B","C","D"], "a": "คำตอบที่ถูกต้อง", "detail": "คำอธิบายกลไกเชิงลึก"},
  {"type": "short", "q": "โจทย์อธิบายกลไก/เปรียบเทียบ", "a": "แนวคำตอบที่ถูกต้อง", "detail": "คำอธิบายเพิ่มเติม"},
  {"type": "long", "q": "โจทย์สถานการณ์วิเคราะห์/คำนวณ (Part A และ Part B)", "a": "วิธีทำแบบ Step-by-step", "detail": "เหตุผลประกอบการตัดสินใจออกแบบ"}
]

"CRITICAL JSON FORMATTING: Do not use raw newlines, carriage returns, or tabs inside JSON string values. If you need to format text, use standard escaped characters like \n."
CRITICAL JSON ESCAPING RULE: 
NEVER use unescaped double quotes (") inside your string values (for questions, options, answers, or details). If you need to quote something, use single quotes (') instead. For example, write 'DMA' instead of "DMA". This is absolutely mandatory to prevent JSON parsing errors.
